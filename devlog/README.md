## 📝 Writing Guidelines

### 良い記録の例
- **具体的**: "Lambda関数がタイムアウト" ではなく "Lambda関数がBedrockコール時に30秒でタイムアウト。原因は〇〇"
- **感情も記録**: "めっちゃハマった..." "これは楽しい！" など
- **再現可能**: コード例、エラーメッセージを貼る

### 避けること
- 抽象的すぎる記述
- 専門用語だけの羅列
- ネガティブすぎる表現
